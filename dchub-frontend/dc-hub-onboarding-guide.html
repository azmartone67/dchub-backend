Onboarding Overview

Single source of truth for DC Hub user onboarding — from free sign-up through Pro activation and account management.

🆓
Free Sign-Up
Self-service, no payment required

User registers → account created → free API key generated → welcome email sent via SendGrid.

⚡
Pro Upgrade
Stripe Checkout + webhook

User pays via Stripe → webhook fires → DB updated to Pro → Pro API key generated → credentials emailed.

🔧
Manual Fallback
When automation fails

Check Stripe logs → verify DB records → create/update in Replit Shell → email credentials to customer.

✅
Verification
Post-activation checklist

User can log in → dashboard shows correct plan → API key works → old keys deactivated.

⚠️ SQLite Persistence Risk

SQLite is file-based and not guaranteed to persist across Replit deployments. #1 risk to customer data. PostgreSQL migration planned.

Pro Plan Upgrade

Stripe Checkout handles payment, then a webhook triggers activation. Most onboarding issues occur here.

Happy Path (Automated)

User clicks "Upgrade to Pro"
Dashboard or pricing page.
Stripe Checkout
Redirected to Stripe's hosted payment page.
Payment completes
Stripe fires checkout.session.completed webhook.
Webhook activates
Looks up user → updates to Pro → generates Pro API key.
Credentials emailed
SendGrid sends API key (10k/day, r/w).
User is live
Dashboard shows Pro. Full access enabled.
🔴 Webhook URL Must Match Deployment

If Replit redeploys and Stripe webhook isn't updated, all activations silently fail. Verify after every deploy.

Webhook Handler Must:

  • Look up user by email from Stripe session
  • Check for duplicate records
  • Update plan → "pro", subscription_status → "active"
  • Store stripe_customer_id
  • Generate Pro API key (read/write, 10k/day)
  • Send welcome email with raw API key
  • Log success/failure

Post-Deploy Verification

  • Stripe Dashboard → Developers → Webhooks
  • Endpoint URL matches current Replit deployment
  • All 4 events subscribed
  • Recent Deliveries show 200 status
  • No old/duplicate endpoints active

Known Issues

Documented onboarding problems with root cause and resolution.

HIGHWebhook Fails to Create User
▼

Symptoms: Customer pays but no account. No email.

Root Cause: Webhook URL stale or handler crashed.

Fix: Check Stripe webhook logs → verify URL → create manually → email credentials.

HIGHDuplicate User Records
▼

Symptoms: Logs in but sees "free" despite paying.

Root Cause: Free sign-up record #1 + Stripe webhook record #2.

Fix: Query all records by email → update login record to Pro → transfer keys → delete duplicate.

FIXEDPassword Hash Mismatch
▼

Symptoms: Can't log in. "Invalid credentials."

Root Cause: Webhook used a placeholder string ('stripe_checkout') instead of calling hash_password(). Manual resets used plain SHA-256 instead of pbkdf2_hmac with salt.

Fix (applied): Webhook now uses hash_password() with pbkdf2_hmac + salt. Temp password is included in the welcome email.

✅ Hashing Standardized

All auth uses pbkdf2_hmac('sha256', password, salt, 100000) → stored as salt:hash. Manual scripts updated to match.

MEDIUMAPI Key Never Delivered
▼

Symptoms: Account shows Pro but no key received.

Root Cause: SendGrid not configured or email failed.

Fix: Generate new key → deactivate old → email to customer.

CRITICALSQLite Database Reset
▼

Symptoms: Multiple customers lose access simultaneously.

Root Cause: Replit deployment reset SQLite file.

Permanent Fix Required

Migrate to PostgreSQL. Single biggest risk to customer accounts.

Quick Diagnosis

Customer reports an issue? Find the symptom below.

🔍
"I paid but I don't have an account"
Stripe webhook failed or endpoint URL stale.
Steps:
1. Check Stripe → Webhooks → Recent Deliveries
2. Update URL if stale
3. Create account manually
4. Email credentials
"My plan still shows Free"
Duplicate user records — free + Stripe-created.
Steps:
1. SELECT * FROM users WHERE email LIKE '%...'
2. Find login record (valid password_hash)
3. Update to plan='pro', add stripe_customer_id
4. Transfer keys, delete duplicate
"I can't log in"
Password hash mismatch or placeholder password.
Steps:
1. Check if last_login is NULL
2. Generate: secrets.token_urlsafe(12)
3. Hash with same algorithm as login route
4. UPDATE password_hash
5. Send new temp password
"I never got my API key"
SendGrid not configured or email failed.
Steps:
1. Check api_keys for existing keys
2. Deactivate old keys
3. Generate new in Replit Shell
4. Email raw key
"How do I manage my subscription?"
Self-service not built yet.
Direct to Stripe Customer Portal if integrated, otherwise handle manually via Stripe Dashboard.
"My API key stopped working"
DB reset, key deactivated, or rate limit.
Steps:
1. SELECT * FROM api_keys WHERE user_id=?
2. Verify is_active=1, plan matches
3. Check calls_today vs limit
4. Regenerate if missing

Manual Account Creation

Emergency process when webhook fails. Run in Replit Shell.

Step 1: Verify Customer Doesn't Exist

python3 -c "
import sqlite3
conn = sqlite3.connect('dc_nexus.db')
c = conn.cursor()
c.execute('SELECT * FROM users WHERE email LIKE ?', ('%CUSTOMER_EMAIL%',))
print([d[0] for d in c.description])
for r in c.fetchall(): print(r)
conn.close()
"

Step 2: Create User + API Key

⚠️ Password Hashing

DC Hub uses pbkdf2_hmac with a random salt (format: salt:hash). Do NOT use plain SHA-256 — logins will fail.

python3 -c "
import sqlite3, hashlib, secrets, datetime
conn = sqlite3.connect('dc_nexus.db')
c = conn.cursor()
now = datetime.datetime.utcnow().isoformat()

# Hash password with pbkdf2_hmac (MUST match app's hash_password)
pwd = secrets.token_urlsafe(12)
salt = secrets.token_hex(16)
h = hashlib.pbkdf2_hmac('sha256', pwd.encode(), salt.encode(), 100000)
password_hash = f'{salt}:{h.hex()}'

c.execute('''INSERT INTO users (email, password_hash, name, plan, role,
  api_calls_today, api_calls_total, created_at,
  stripe_customer_id, subscription_status)
  VALUES (?,?,?,'pro','pro',0,0,?,'','active')''',
  ('EMAIL', password_hash, 'NAME', now))

uid = c.lastrowid
key = 'dchub_' + secrets.token_urlsafe(32)
kh = hashlib.sha256(key.encode()).hexdigest()
c.execute('''INSERT INTO api_keys (user_id, key_hash, key_prefix,
  name, permissions, rate_limit_tier, is_active, created_at,
  usage_count, plan, calls_today, calls_total)
  VALUES (?,?,?,?,'[\"read\",\"write\"]','pro',1,?,0,'pro',0,0)''',
  (uid, kh, key[:12], 'NAME Pro Key', now))

conn.commit(); conn.close()
print('User ID:', uid)
print('API Key:', key)
print('Temp Password:', pwd)
"
Replace Placeholders

Swap EMAIL and NAME with actual values. USER_ID is now auto-captured via lastrowid.

Step 3: Reset Password (Existing User)

python3 -c "
import sqlite3, hashlib, secrets
conn = sqlite3.connect('dc_nexus.db')
c = conn.cursor()

# Hash with pbkdf2_hmac + salt (MUST match app)
pwd = secrets.token_urlsafe(12)
salt = secrets.token_hex(16)
h = hashlib.pbkdf2_hmac('sha256', pwd.encode(), salt.encode(), 100000)
password_hash = f'{salt}:{h.hex()}'

c.execute('''UPDATE users SET password_hash=?, plan='pro',
  role='pro', subscription_status='active'
  WHERE email=?''', (password_hash, 'EMAIL'))
print('Updated:', c.rowcount)
conn.commit(); conn.close()
print('Temp password:', pwd)
"
